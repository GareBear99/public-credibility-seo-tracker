## Purpose

The `dev.to-post-tracker` repo is the dedicated evidence and operations repo for DEV.to publishing.

It should track:

- Markdown article sources
- draft/published state
- canonical GitHub links
- public DEV.to URLs
- tags used
- article-to-repo mapping
- cross-link requirements
- forum/social reuse copy
- next-post queue

## Why it matters

DEV.to articles create long-form explanations for projects that are too complex to explain in a short directory listing.

This strengthens:

- Google indexing
- GitHub repo context
- public credibility
- shareable launch links
- forum post support
- cross-linking across the ARC / TizWildin ecosystem
