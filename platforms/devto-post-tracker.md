## Purpose

The DEV.to route acts as the long-form public explanation layer for the ecosystem.

Each post should:

- explain one project clearly
- link the canonical GitHub repo
- use searchable tags
- include a strong description
- point readers toward related ARC / TizWildin repos where useful
- create a public article URL that can be reused in forum posts, KVR/JUCE posts, Facebook groups, Reddit, and directory submissions

## Post tracker repo role

`dev.to-post-tracker` should store:

- article source Markdown
- publish status
- canonical URLs
- final public DEV.to URLs
- cross-link notes
- profile optimization notes
- post cadence / next posts
- social/forum reuse copy

This repo (`public-credibility-seo-tracker`) should link to `dev.to-post-tracker` as the dedicated long-form article evidence source.

## Operating rule

Do not treat DEV.to as a replacement for directories.

Use it as the article hub:

```text
GitHub repo
→ DEV.to article
→ forum/community post
→ directory profile
→ backlink loop
```
